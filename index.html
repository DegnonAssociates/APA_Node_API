Introduction

This documentation is an overview for using the APA membership API.

Allowed HTTPS requests

GET:         Get a resource or list of resources
POST:       Add a new resource. Can be used for update requests for WordPress compatibility. Also used for authentication requests
PUT:        Update a resource
DELETE: Delete a resource

API Base URI: https://www.academicpeds.org/apps/api/v1

Authentication

Most requests require an authentication token be present in order to resolve. In order to obtain an authentication token, a user will send a POST request to 

https://www.academicpeds.org/apps/api/v1/authenticate
and pass the following JSON object 
{ "username":"{username}","password":"{password}"}
If successful, the API will return a JSON object 
{

        "message": "success"     //or "Input not valid" for a failed request

        "token" : {generatedAuthToken}

        "email" : {userEmail}

        "display_name" : {userFirstName}
}
The token can then be passed with future HTTPS request in the header as "x-access-token".

Members resource

The members resource contains all member demographic information. Member information CRUD operations are handled by this resource.

GET

The GET operation is used to view a single or multiple member resources.

To get a list of all member resources, enter the request 

https://www.academicpeds.org/apps/api/v1/members
The request will return the first 25 resources found. Pagination is available by adding a "page" variable to the URL query string.

To get an individual member resource, enter the request https://www.academicpeds.org/apps/api/v1/members/:memberId



        
The member object model is listed below
          
{
            "memberId" //resource ID. Used for accessing individual resources
            "firstName"
            "lastName"
            "middleName"
            "degree" //resource professional degree
            "academicTitle"
            "department"
            "institution"
            "busPhone"
            "email"
            "address"
            "address2"
            "city"
            "state"
            "zip"
            "country"
            "fax"
            "memberType"
            "duesYear" //latest current membership renewal year
}
 


        
POST/PUT

        
An individual Member resource can be created by issuing a POST request to 
          
https://www.academicpeds.org/apps/api/v1/members
 The firstName, lastName, and Email object keys are required, all other keys are optional.

        
An existing member resource can be updated by issuing a POST or PUT request. NOTE: only the owner of the Member resource or an ADMIN user can update an existing resource. This authorization is handled by the provided authentication token. 
https://www.academicpeds.org/apps/api/v1/members/:memberId


        
DELETE

        
An individual Member resource can be deleted by issuing a DELETE request to 
        
https://www.academicpeds.org/apps/api/v1/members/:memberId

      NOTE: only the owner of the Member resource or an ADMIN user can delete an existing resource. This authorization is handled by the provided authentication token.


      
Acitivty resource

      

      
All member activities are handled by this resource. This includes committee memberships, positions held in the organization, etc.
